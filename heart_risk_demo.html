XAI Methods Used in this System
SHAP (Global + Local)
Game-theoretic approach that assigns each feature a contribution value for specific predictions. Guarantees consistency and local accuracy. Used for both global feature ranking and per-patient explanation.
LIME (Local Surrogate)
Creates a locally faithful linear model around each prediction. Provides human-readable "if this value was lower, risk would drop" statements. Ideal for physician communication.
Counterfactual Explanations
Answers "what minimal change would flip the prediction?" Directly actionable: "If cholesterol drops below 220, risk changes from HIGH to MEDIUM." Critical for patient counseling.

DQA Framework — 5 Dimensions
1
Completeness — Checks for missing values; applies smart imputation using population median with uncertainty flagging.
2
Validity — Validates clinical plausibility ranges (e.g., cholesterol 100–600 mg/dL) per medical standards.
3
Consistency — Cross-field coherence checks (e.g., age vs max HR, BP vs medication context).
4
Plausibility — Statistical outlier detection using IQR-based scoring against UCI Heart Disease distribution.
5
Temporal Stability — Flags values that are physiologically unusual given patient age and clinical context.

↓ Explanations + Confidence
🏥
Layer 4: Clinical Decision Support Interface
Risk dashboard · Physician-readable XAI reports · Actionable recommendations · Counterfactual "what-if" explorer · Audit trail · EHR integration API
How We Differ from Existing Work
✦
DQA as First-Class Citizen — Unlike existing systems where data quality is a preprocessing step, our architecture treats it as a continuous, integrated layer that influences every downstream component.
✦
Quality-Weighted Ensemble — Model confidence weights dynamically adjust based on DQA score. Noisy data → lower ensemble weights → wider uncertainty bands. No existing paper does this.
✦
Triple XAI Stack — SHAP + LIME + Counterfactuals together, not post-hoc but architecturally embedded. Results are calibrated against data quality.
✦
Clinical Alignment — Explanations are translated into physician-readable language. System flags when to override AI due to data quality concerns.

Dataset Used
📁
UCI Heart Disease Dataset — 303 patients, 13 clinical features, binary classification (presence/absence of heart disease). Cleveland clinic foundation data.
🔄
Train/Test Split — 80/20 stratified split with 5-fold cross validation for robust evaluation.
📈
Evaluation Metrics — Accuracy, AUC-ROC, F1-Score, Sensitivity (Recall), Specificity, Brier Score for calibration.
